Automation & Cypress FundamentalsThis repository contains research, implementation examples, and basic test suites covering core automated testing concepts using Cypress.What I LearnedDuring this assignment, I researched and practiced the foundational blocks of Cypress automation, including:Test Structure: Organizing automated scripts into logical suites and blocks using BDD syntax hooks.Locators & Interaction: Finding and targeting elements reliably using optimal attribute strategies.Assertions: Validating page behaviors, element visibilities, states, and text contents dynamically.Advanced Control: Handling complex UI conditions like dynamic waiting, file handling, network interception, and managing page object layouts.ReflectionWhat was the easiest concept to understand?The easiest concept to understand was organizing tests using standard structures like describe() and it(). They provide immediate logical readability to individual tests and test suites. Basic actions like cy.visit() and cy.click() were also highly intuitive because they mirror straightforward human behaviors when interacting directly with a standard web browser page.What was the most challenging?The most challenging aspect was configuring asynchronous interactions properly, such as dealing with network traffic via cy.intercept() or managing multi-step element lookup scoping rules. Dealing with complex components like dynamic calendar dropdown pickers or file upload overrides required a deeper understanding of asynchronous control flow and DOM rendering cycles.What would you like to learn next about Cypress?I would love to learn more about setting up Continuous Integration (CI) test execution runs using tools like GitHub Actions. Additionally, exploring parallel cross-browser execution setups and visual regression snapshot comparison techniques would be fantastic milestones for expanding this framework.
